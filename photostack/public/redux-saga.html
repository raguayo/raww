What is Redux Saga?

  • redux-saga is a library that aims to make handling side-effects (i.e. asynchronous things like data fetching) on React/Redux applications easier and better.
  • A saga is like a separate thread in your application that's solely responsible for side effects.
  • ES6 Generators are used to make your async code look more synchronous making it easier to read, write and test.

Redux Saga Watchers & Workers

  • The main saga file is usually split into two different types of sagas: workers & watchers
  • Watcher saga's see every action this is dispatched to the redux store, if it matches the action they are told to handle, they assign it to their worker saga

Saga Helpers

  • takeEvery - takes eery matching action and runs the instructed worker saga, can run concurrent tasks
  • takeLatest - takes every matching action and runs the instructed saga but cancels any previous saga tasks if still running

Redux Saga Advantages

  • Synchronous looking code (hides away asynchronicity)
  • Easy to write test with little to no mocking (for asynchronous code by checking values as iterating through generated function and yielding)
  • Sagas are composable (*yield)
  • Action creators are pure
  • Isolated side-effect code to a single area of the application (sagas.js)
  • Many helper functions & solid documentation
